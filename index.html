What is a Side-Channel Attack?

Traditional vs Side-Channel

Traditional cryptanalysis attempts to break the mathematical structure of a cipher — requiring enormous computational power. For AES-128, brute-force would need 2¹²⁸ ≈ 3.4×10³⁸ operations.

Side-channel attacks exploit physical information leaked during computation. Rather than attacking the algorithm, they attack the implementation. CMOS transistors draw more current when switching from 0→1 than 1→0, creating data-dependent power signatures.

Attack Flow

1

Measure Power

Connect oscilloscope or current-sense resistor. Record N power traces during AES encryption of known plaintexts.

2

Compute HW Model

For each of 256 key hypotheses, compute Hamming Weight of SubBytes(plaintext ⊕ k_hyp).

3

Correlate & Recover

Compute Pearson correlation. The hypothesis with highest |r| at the leak time-point IS the correct key byte.

Why AES SubBytes is Vulnerable

Known Input

Controlled Plaintext

The attacker encrypts chosen plaintexts, making the input to SubBytes(p ⊕ k) known for any key hypothesis. This creates an exploitable data-power relationship.

High HW Variance

S-Box Nonlinearity

AES S-Box output has Hamming Weights distributed 0–8 with large variance. This amplifies the power signal, making it detectable even through noise.

First Round

Direct Key Contact

The first SubBytes operates on (plaintext ⊕ key). Only 256 hypotheses exist per byte. Each byte is independent → 16 separate 256-search attacks.

Full Source Code

aes_sim.py — Core Simulation

# ─── AES S-Box ──────────────────────────────────────────────────────────── SBOX = [ 0x63,0x7c,0x77,0x7b,0xf2,0x6b,0x6f,0xc5,... # 256 entries (FIPS 197) ] def hamming_weight(x: int) -> int: """Count 1-bits — models CMOS power consumption""" return bin(x).count('1') def subbytes_output(plaintext_byte: int, key_byte: int) -> int: """AES SubBytes: target of the CPA attack""" return SBOX[plaintext_byte ^ key_byte] def simulate_power_traces(plaintexts, true_key, noise_std=0.5, time_points=50): traces = rng.normal(0.0, noise_std, size=(N, time_points)) for i in range(N): hw = hamming_weight(subbytes_output(plaintexts[i,0], true_key[0])) traces[i, leak_time] += hw # inject signal at leak_time return traces def correlation_power_analysis(plaintexts, traces, byte_idx=0): correlations = np.zeros((256, T)) for k_hyp in range(256): # Power model: HW of predicted SubBytes output hw_model = [hamming_weight(subbytes_output(p, k_hyp)) for p in plaintexts[:,byte_idx]] # Pearson correlation with each time sample corr = (traces - traces.mean(axis=0)) * (hw_model - hw_mean)[:,None] correlations[k_hyp] = corr.mean(axis=0) / (hw_std * tr_std) best_guess = np.max(np.abs(correlations), axis=1) return correlations, best_guess

Academic Report Summary

Abstract

Side-channel attacks represent a category of cryptanalytic threat that bypasses the mathematical hardness of symmetric encryption by exploiting physical information emanating from computing devices during cryptographic operations. This project presents a complete simulation of Correlation Power Analysis (CPA) against the AES-128 cipher implemented in Python. Using the Hamming Weight leakage model of the SubBytes S-Box operation, we demonstrate that the 128-bit secret key can be fully recovered using as few as 50 power measurements under realistic noise conditions (σ = 1.2), achieving 100% success across 15 independent Monte Carlo trials. The attack reduces the AES-128 keyspace from 2¹²⁸ to 16 × 256 = 4,096 hypothesis evaluations. We further analyse the effect of noise level and trace count on attack success rate, and evaluate three classes of countermeasure: noise injection (hiding), Boolean masking (algorithmic), and clock jitter (temporal randomisation).

Conclusion

This project demonstrates that CPA is a practical and highly efficient attack against AES-128 on unprotected hardware. The dramatic reduction of the effective keyspace — from 2¹²⁸ to 16 × 256 — confirms that cryptographic security is inseparable from physical implementation security. Countermeasures including Boolean masking, dual-rail logic, and random clock jitter each degrade attack performance but none eliminates the threat in isolation. Defence-in-depth is mandatory for production-grade secure devices. The simulation framework developed here provides a pedagogically complete, reproducible environment for further SCA research without requiring physical hardware.

Future Work: Extension to AES-256, higher-order CPA on masked implementations, hardware validation on ChipWhisperer Lite platform, and integration with profiling attacks (Template Attack, Deep Learning-based SCA using neural networks trained on trace datasets).
